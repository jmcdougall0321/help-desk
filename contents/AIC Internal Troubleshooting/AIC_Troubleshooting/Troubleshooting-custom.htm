Troubleshooting – “Can I turn on, hide, etc. this field?”

Clients like to customize their systems, they often reach out asking to hide/show/etc. certain fields in the application. Let’s take a look at how you can determine if/where you can do this (hint- for the most part, the answer is yes!)…

 

 

Step 1 – What is the client asking for?

  • To turn on something that they don’t see
    • Example “do you have a section for tracking drug testing that we can use?”
  • To hide a field
    • Example “can we hide the severity consequence matrix?”
  • To make a field required
    • Example “we want to require incident level” Let’s take a look!

       

      Step 2 – Where can we do these things?

  • There are 3 main areas we can check for these type of requests:
    • Form actions in the profile security
      • This will accommodate a variety of things, most things are available here so I recommend you check here first (things like requiring fields, hiding fields, turning on sections of the form, etc.)
    • Customize module
      • Customize not only enables you to add new fields on forms but it also can hide/read-only/require fields. This depends on if the form is available to customize. Currently the forms that have this availability are as follows below. Please note, any form that does NOT have customize does not have any of this functionality, this is ONLY for these forms:

 

 

        • General Details within all incident modules
        • Corrective action input form across all modules
    • Variables
      • While this is the ‘oldest’ option, there are still a lot of valuable system customization options in here. The variables are located under aic tools > variables. This grid is somewhat complex to search so I recommend searching only a small part of what you are looking for in the variable name column. Sometimes you have to do a few searches to see if anything is available using different keywords.
        • For example, if you were looking for ‘drug and alcohol test’, try to search ‘drug’ then try ‘test’. Eventually you will determine if it is available or not by searching here.
        • This is typically the last resort, use the profile first
    • Turn on/Turn off module item
      • If you cant locate something in the profile, it may simply not be turned on in the system. AIC controls what is on and what is off through the aic tools module > turn on turn off module item (known as TOTO).
      • Within TOTO, you can update each module, and search for form actions that are not turned on.
        • Checked = turned on, available in the profile
        • Unchecked = turned off, not available in the profile
      • You might be wondering, why don’t we just turn everything in TOTO on?
        • It is based on what the client licenses.
        • Many of the available things DO NOT WORK. Take caution with the grid buttons and reports as some of these are turned off for a reason. If in doubt, just check with someone!

          Step 3 – It’s not available currently:

  • Let the client know it is not available.
  • If they push back asking us to add it after you let them know it isn’t available, add it to the potential future enhancements list (check first to see if anyone else has added it to avoid duplicates).
