Anytime an alert is created the logged in user will automatically inherit that alert and will become a participant. If you do not wish to receive notifications for all the alerts you’ve created, best practice is to create a default “dummy” account. This would then allow you to login through the dummy account and create alerts without having to receive notifications for every single one. AIC can also update the created user of your current active alerts to the dummy account once it has been created. You will remain an owner of these alerts so that you can edit them as needed through your normal administrator account.

 

An example of a dummy account is shown in the screenshot below. You would want to make sure that this account has access to all locations so that you can assign alerts out to your divisions accordingly. This can be accomplished by applying an Administrator profile to this user record. You would also want to insert a fake email address.

Once the account is setup, I can create a development task to have the alerts that you created reassigned.

 

When an alert is COPIED the person who copies the alert becomes the creator and the creator of the original alert would not inherit the notifications.
