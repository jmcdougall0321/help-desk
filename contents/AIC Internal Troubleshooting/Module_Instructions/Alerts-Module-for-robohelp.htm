Alerts Module

Alerts Module Overview

In the Alerts Module, you are able to set up multiple alerts across all Incident Types and Modules.

Depending on the Module chosen, different alert types become available to set up.

  • For most alert types to go out, the Responsible Role for each User needs to be selected for that Module. This is done at the User Profile Level.
  • There are Supervisor/Admin Alerts that give Users the ability to filter and select the recipients that the alert will go out.

 

Alerts Module – Alerts Module Item

 

Adding a Alert Record:

  • Go to the Alerts Module, Alerts module

  • Select the ‘Add’ button
  • Once the Add button has been selected the ‘Add Alert Form will become available.
  • Category: This is the first required field in which the Category (Module) will need to be selected.
    • The different Modules that populate depend on User Access.
  • Type: This is the second required field, that will populate with all Alert Types available for that Module.
    • Examples: Corrective Action Coming Due
    • Corrective Action Coming Due Reminder
    • Corrective Action Completed
    • Corrective Action Past Due
    • Corrective Action Past Due Reminder
    • Corrective Action Responsible Party

 

Step 3:

  • Selecting the Type of Alert will determine the flexibility of the alert set up.

     

  • Please see the example below: Corrective Action Coming Due : This alert is a alert for a Supervisor/Administrator of the application.

 

The Administrative qualities of the below alert allows you to set up the following:

    1. Setting up the amount of days you would like to be notified before a Corrective Action is due
    3. Setup Recipients to receive this alert. This can be done by Profile or Last Name.

 

Step 5:

The General Set up of the below alert is as follows:

  • This Alert will go out to ALL People who have been assigned a Corrective Action in the Category (Module) that it is being entered for.
  1. Description – Title of the Alert
  2. Notes – This can be instructions on next steps in completing the Corrective Action.

 

TYPE: CORRECTIVE ACTION COMING DUE

SCREENSHOT

DESCRIPTION:

 

This alert is for someone such as a supervisor who wants to see a list of all the corrective actions coming due at their facility or facilities anywhere from 1-30 days in advance of the due date. This can be set up to send to a profile or a list of individuals. This alert is for corrective actions added in the Activity Calendar that are not related to another module or incident such as Injury, Fires, Spills etc.

 

 

 

 

TYPE: CORRECTIVE ACTION COMING DUE REMINDER

SCREENSHOT

DESCRIPTION:

This sends the individual responsible person an alert that they have a corrective action coming due anywhere from 1-30 days in advance of the due date. This alert is for corrective actions added in the Activity Calendar that are not related to another module or incident such as injury, fires, spills etc.

 

 

 

 

 

TYPE: CORRECTIVE ACTION COMPLETED

SCREENSHOT

DESCRIPTION:

This alert is for someone such as a supervisor who wants to be alerted to all the corrective actions marked completed at their facility or facilities. This can be set up to send to a profile or a list of individuals. This alert is for corrective actions added in the Activity Calendar that are not related to another module or incident such as Injury, Fires, Spills etc.

 

 

 

 

 

TYPE: CORRECTIVE ACTION PAST DUE

SCREENSHOT

DESCRIPTION:

This alert is for someone such as a supervisor who wants to be alerted of all the corrective actions past due at their facility or facilities anywhere from 1-30 days past the due date. This can be set up to send to a profile or a list of individuals. This alert is for corrective actions added in the Activity Calendar that are not related to another module or incident such as Injury, Fires, Spills etc.

 

 

 

 

 

TYPE: CORRECTIVE ACTION PAST DUE REMINDER

SCREENSHOT

DESCRIPTION:

This sends the individual responsible person an alert that they have a corrective action past due anywhere from 1- 30 days after the due date. An email will be sent daily after that until the corrective action is marked complete. This alert is for past due corrective actions added in the Activity Calendar that are not related to another module or incident such as Injury, Fires, Spills etc.

 

 

 

 

 

TYPE: CORRECTIVE ACTION RESPONSIBLE PARTY

SCREENSHOT

DESCRIPTION:

This sends the responsible person an alert that they have been assigned a corrective action in the Activity Calendar with the details and due date.

 

 

 

 

TYPE: INJURY RECORD CREATOR

SCREENSHOT

DESCRIPTION:

This sends an email alert to the individual who entered the record acknowledging the injury has been added to the system.

 

 

 

 

TYPE: NEW SUBMITTED CLAIMS

SCREENSHOT

DESCRIPTION:

This alert sends an email to specified users that an Injury/ Illness claim has been submitted from the application to your company's TPA. This alert can be sent as a Daily summary with all of the submitted claims from the Incident Reporting Module for that day.
