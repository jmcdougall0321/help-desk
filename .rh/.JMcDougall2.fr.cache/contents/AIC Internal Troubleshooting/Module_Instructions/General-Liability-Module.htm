General Liability Module   General Liability Module Overview The General Liability Module is an extension of our Incident Management suite. This Module allows for quick entry of streamlined data, automated email alerts and messaging, and reporting capabilities of GL Claims. In this Module you will enter of the information that you have gathered regarding a General Liability Incident. General Liability shares a common user interface with our other Incident Management Modules. When adding or editing a record there will be a series of tabs across the top of the incident form that will contain required data fields to save or maintain the record and a series of tabs across the bottom of the incident form for collecting additional information or attachments.   General Liability Module Item: Within the General Liability Module Item you will see a dynamic search and display grid showing all previously entered incidents in the General Liability Module. Our search grid is unique in that by selecting the heading at the top of each column you can sort the records alphabetically or by date in the column selected. You can also search for a specific record using multiple columns to narrow down your search. Our system also supports wild card searches by putting a % symbol in front of and behind the partial word you have entered (ex: %Diego%). Adding a General Liability Incident Record: When adding a new record follow the steps below: Select the Add button, the Incident Location Form will populate All required fields (marked with a red asterisk) must be complete before you can move onto the next step. You can narrow down your location search by first choosing from the Select Incident Organization Level drop down. Once you find the group or level you are looking for click on it. All the locations available will automatically populate in the Select Incident Location drop down. Select the location for this record. Once the Location is selected you will need to answer three questions related to the nature of the General Liability Incident which will determine what Main Tabs are available to you on the General Liability input form Work Location - Defaulted to Location that was selected in previous Location. This location can modified by using the ‘Edit’ button to change the Incident Location. Claim Number - Field available for your Worker’s Comp TPA. This can be manually entered or populated by an automated feed. Case Status - Dropdown field available to determine the status of Case (example: Open, Closed, Under Review) Major Event - Optional Data field that is used for capturing general incident information and allocating it across multiple modules to associate common dates, times, location and general details of a major event. Location Code - Additional unique identifier used when submitting claims. Claim Type - Identifying if this claim will be a 1st Party Vehicle or 3rd Party Vehicle Incident Level - Pick list that is client defined, as are all pick list values throughout the application. The “what’s this” button can be selected to display your incident severity levels and their corresponding descriptions Severity Consequence Matrix - A pick list value displays High, Medium, and Low for selection to determine by the PDF Matrix given to AIC across all Incident Modules. Note: Each Matrix can be different for each Incident Type Accident Occur on Employer Premises? - Yes/No Radio Button Was this incident preventable? - Yes/No Radio Button   Date of Incident - Required Field; has a calendar control for you to quickly select the day that the incident had occurred, however you can also manually enter in the Incident Date using the following format: mm/dd/yyyy Incident Time - Dropdown value for selection of the Time the Incident occurred; however, you can also manually enter in the number(s) within the dropdown to quickly select the accurate timeframe. Date Reported to Employer - Required Field; has a calendar control for you to quickly select the day that the incident was reported, however you can also manually enter in the Incident Date using the following format: mm/dd/yyyy Time Reported to Employer - Dropdown value for selection of the Time the Incident was Reported; however, you can also manually enter in the number(s) within the dropdown to quickly select the accurate timeframe. Work Shift – Pick list that is client defined to identify which shift the incident took place on. Part of Shift – Pick list available to be selected for Before, During and After. Preparer First Name - Auto populated based on logged in User. Preparer Last Name - Auto populated based on logged in User. Preparer Phone - Auto populated based on logged in User. Preparer Title - Auto populated based on logged in User. Preparer Information is the same as Contact? - Yes/No radio button that can be selected. This filed defaults to YES however if selected NO, there are 4 new required fields that populated to fill out (Contact First Name, Contact Last Name, Contact Phone, and Contact Title) Work Station/Area - pick list value is also made up of client defined values. This dropdown is tied to the Work Shift Module Item within the Administration Module. With the [ADD NEW] form action turned ON, you can also have the option to add a new value to this dropdown on the fly. Specific Location – An optional text box to describe more specifically where this Incident occurred. Give a Brief Description of the Incident – A text box that allows for additional information related to the incident (250 character limit). Employee and/or Contractor description of what happened – Additional optional text box available for an Employee/Contractor statement (1000 character limit).   Claimant Details Main Tab This Main Tab will capture the Claimant (injured non employee) Details of the General Liability Claim, and the Claimant Employer Details.   Bodily Injury Details Main Tab This Main Tab will only show if “Yes” is selected for the following question, “Was there an Injury as a Result of this Incident?” This tab will capture the Treatment Information, Key factors, and Descriptions of What Happened in Detail.   Property Damage Details Main Tab This Main Tab will only show if “Yes” is selected for the following question, “Was there Property Damage as a Result of this Incident?” This Tab will capture the Property Damage Details including the Type of Property Damage and associated cost of the damage.   Product Liability Details Main Tab This Main Tab will only show if “Yes” is selected for the following question, “Is this a Product Liability Claim?” This Tab will capture the Product Liability Details. Root Cause Main Tab This Main Tab will capture the Incident Details of the General Liability Claim. This tab allows the user to set up a series of Why Statements and Answers to determine the Root Cause of the General Liability Incident. Causal Factors Main Tab The Causal Factors Main tab allows you to use a series of text boxes and pick lists to assist in the investigation of the Immediate and underlying causes of this incident. What Happened Mini Tab: This is a text field used to give a description of the incident in greater detail due to the larger amount of text than allowed in “Give a Brief Description” text box. Updates Mini Tab: The Updates Mini Tab provides an electronic diary for tracking notes, all of which are date stamped and user stamped. Corrective Actions Mini Tab: Anywhere in the system you can assign a corrective action that are enforced through a series of automated emails and alerts. The controls that govern the coming due, escalation and past due emails and alerts are client defined within the Alerts Module and Alerts Module Item. Once you have completed your initial set up, they will fire off automatically as applicable. All corrective actions can be updated directly from an email, you do not have to log into the system to update the statuses.   Cost/Hours Mini Tab: This tab provides the user the ability to view any cost information associated with this incident. The cost information can be pulled back from the TPA and populated here automatically, or can be entered manually based on the users profile level. Additionally, the reserve for this incident can be assigned and tracked within this mini tab. People Involved Mini Tab: This is where information from a witnesses, first responders, or any other person/people that may be associated with the incident can be captured. Record who they are, how to get a hold of them, and their statement.   History Mini Tab: This feature will show you any/all other incidents the employee has been involved in with your organization since the start of their employment. You can view the historical incident details by selecting the incident record in the display grid and select the “View” button. Files Mini Tab: This is where supporting documentation regarding this incident can be attached. Attachments can include pictures, videos, scanned documents, pdf’s and more. By selecting the “View Valid File Types” button, you can see all file formats that are supported in this tab.   Review Approval Process Mini Tab: This is where you can set up a review/ approval process, using a series of email alerts and reminders, for an incident to go through the different levels of review necessary with an automated process. The review process is set up in the Administration Module within the Review Process Module Item. Communication Mini Tab: In addition to automated email alerts you can broadcast an incident report to a group of people via an email, directly from this tab. Select the person or group of people you would like to receive the email and attach files from the “Files” mini tab using point and click check boxes. To set up the names or groups who can receive these emails are done in the Administration Module within the Communications Module Item. The communications is set up based on location. Once a location is selected the list of recipients can be created by individuals which can then create groups. To upload individuals there is an Upload Wizard button which gives you step by step directions to load more than one person at a time.   Edit, Delete, View, Print and Search a General Liability Record Edit - To edit a General Liability record follow the steps below: Highlight desired General Liability record with one click Select the Edit button at the top Edit the General Liability record as needed Select Save and Close   Delete – To remove a General Liability record, we usually leave all records for historical purposes even if they are no longer needed at this point in time, follow the steps below: Highlight desired General Liability record with one click Select the Delete button at the top Select the Delete button within the popup   Print – To print a General Liability record follow the steps below: Highlight desired General Liability record with one click Select the Print button at the top A PDF version of the fire record will be created Select print to print the record   View – This will open up the General Liability record so you can view the details without editing them: Select desired General Liability record Click the View button The record will open allowing you to view the details   Send to Excel – This will take all the General Liability records shown in the search grid and transport it to an excel sheet.
